# CLAUDE.md — Портфолио Gleb Paikachev

Контекст проекта для работы в Claude Code. Положи этот файл рядом с `index.html` — Claude Code читает `CLAUDE.md` автоматически.

## Что это

Одностраничный сайт-портфолио: документальная фотография + политическое искусство (артивизм). Тёмная тема, золотой акцент, журнальная типографика.

## Стек

- **Один файл** — весь сайт в `index.html`. Никакой сборки, npm, фреймворков.
- Чистые HTML + CSS + JS. CSS целиком в `<style>` внутри `<head>`. JS целиком в одном `<script>` перед `</body>`.
- Шрифты — Google Fonts (тянутся из интернета): Bebas Neue, Spectral, Space Mono, Cormorant Garamond.
- Запуск: открыть `index.html` в браузере двойным кликом. Сервер не нужен.

## Как править без боли

Не ориентируйся на номера строк — они сдвигаются. Указывай место по HTML-комментарию (`<!-- FEATURED -->`) или по классу/id. Все секции помечены комментариями-якорями.

Перед крупными правками делай копию файла.

## Дизайн-система (CSS-переменные в `:root`)

Цвета:
- `--black` `#0c0c0c` — основной фон
- `--dark` `#141414` — фон вторичных блоков
- `--gold` золотой акцент, `--gold-dim` приглушённый, `--gold-faint` едва заметный
- `--white` `#f0ece2` — текст, `--grey` `#888580` — вторичный текст, `--mid` `#555250` — границы

Шрифты (через переменные):
- `--bebas` (Bebas Neue) — крупные заголовки
- `--spectral` — основной текст
- `--mono` (Space Mono) — технические подписи, метки, номера
- `--fraunces` (Cormorant Garamond) — курсивные акценты, цитаты

Используй переменные, а не хардкод цветов — так тема остаётся цельной.

## Структура (порядок секций в DOM)

1. `<nav>` — фиксированная навигация сверху. Пункты: About, Work, CV, Skills, Contact.
2. `.cursor` / `.cursor-ring` — кастомный золотой курсор (только десктоп).
3. `<!-- HERO -->` `.hero` — два столбца: текст слева, фото-плейсхолдер справа с диагональным разделителем.
4. `<!-- MARQUEE -->` — бегущая строка с направлениями работы.
5. `<!-- ABOUT -->` `#about` — текст слева + блок фактов справа.
6. `<!-- FEATURED -->` `#featured` — две вкладки:
   - `#panel-photography` (`#photography`) — сетка документальных фото.
   - `#panel-artivism` (`#artivism`) — список арт-проектов (заголовок слева + проекты справа).
7. `<!-- EXPERIENCE -->` `#experience` — горизонтальный таймлайн карьеры.
8. `<!-- SKILLS -->` `#skills` — три колонки навыков.
9. `<!-- CONTACT -->` `#contact` — золотая секция с контактами.
10. `<footer>`.

## Интерактив (что делает JS)

- **Вкладки Featured** — клик по `.featured-tab` переключает панель и заново проигрывает анимации появления.
- **Кастомный курсор** — золотая точка + кольцо, плавно догоняющее. Запускается только на устройствах с мышью (`matchMedia('(hover: hover)')`), на тач отключён.
- **Scroll reveal** — элементы с классом `.reveal` плавно проявляются при попадании в экран (IntersectionObserver).
- **Таймлайн** — наведение на `.exp-card` показывает всплывающую подсказку с описанием; таймлайн можно таскать мышью (drag-to-scroll).
- **Фото-карусели** — в фото-ячейках стрелки ←/→ листают слайды. Ячейка с атрибутом `data-auto-carousel` листается сама каждые 3.2 сек.
- **Описания проектов** — наведение на фото-ячейку с `data-project-description` подменяет текст в текстовой ячейке-подсказке (только десктоп/hover).
- **Лайтбокс** — клик по миниатюре `.art-img-thumb` открывает фото на весь экран. Закрытие — клик или Esc.

## Как добавлять контент

### Реальные фото в документальную сетку
Ячейки `.photo-cell` сейчас плейсхолдеры (иконка ⬡ + номер). Чтобы вставить фото — добавь `<img>` внутрь `.photo-inner`. JS сам определит вертикальные снимки и применит правильный `object-fit`.

### Фото в арт-проекты
Готовый API в JS:
```js
addProjectImages('rave-not-raid', ['url1.jpg', 'url2.jpg']);
```
Доступные id проектов: `hundred-year-man`, `rave-not-raid`, `kanonerskiy-walk`, `chernobyl-prayer`, `care-tent`, `barents-eco-art`.

### Новая карточка опыта в таймлайн
Шкала: **2006–2026 = 100%**. Для карточки задаёшь inline-стиль:
- `left%` = (год_начала − 2006) / 20 × 100
- `width%` = длительность_в_годах / 20 × 100
- ряд над осью: `bottom: 14px / 110px / 210px`; под осью: `top: 14px / 110px`

Рядом добавляешь `.exp-stem` (вертикальная ножка) и `.exp-dot` (точка на оси) с теми же left%. Данные для подсказки — в атрибутах `data-role`, `data-org`, `data-dates`, `data-desc`.

## Подводные камни (НЕ сломай)

- **`body { overflow-x: clip }`** — именно `clip`, не `hidden`. `hidden` ломает все `position: sticky` на странице. Это специально.
- **`.featured-tabs` sticky `top: 3.75rem`** — прилеплены под фиксированной навигацией. Если меняешь высоту `<nav>`, синхронизируй этот `top`.
- **Сетка документальных фото** — позиции ячеек захардкожены в CSS через `.photo-cell:nth-child(N)` в процентах, рассчитаны из макета 1280px. Высоту сетки задаёт `padding-top: 63.5%` у `.photo-grid-inner`. Если меняешь СОСТАВ или ПОРЯДОК фото — придётся пересчитывать координаты. На планшете (≤1024px) сетка переключается на обычный flow-grid, поэтому хардкод там не мешает.
- **Таймлайн** — координаты карточек заданы inline в процентах по шкале. Минимальная ширина 1200px, дальше горизонтальный скролл.
- **Курсор** — на фото и кнопках стоит `cursor: none`, потому что есть кастомный. На тач он скрыт через `@media (hover: none)`.
- Плейсхолдеры под замену: иконки ⬡, текст «Your portrait here» в hero, золотые градиенты вместо фото.

## Адаптив

Два брейкпоинта:
- **≤1024px** — одноколоночные раскладки; hero-фото и бегущая строка скрыты; фотосетка → flow-grid 2 колонки; левый sticky-заголовок артивизма становится статичным.
- **≤640px** — пункты меню скрыты (остаётся имя); фотосетка в 1 колонку.
